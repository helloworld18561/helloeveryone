# 자동 보관 구조 설계

이 문서는 정전, 재불팅, 프로그램 오류 발생 시 자동으로 시스템을 안전하게 보관하는 구조를 설계하는 방법을 정리합니다.

## 1. 전원 보관
정전 후 전원이 돌아오면 장비가 자동으로 켜여진도록 UPS 설치와 BIOS 설정을 합니다. `Restore on AC Power Loss` 옵션을 'Power On'으로 설정합니다.

## 2. 불팅 후 프로그램 자동 실행
서비싱 등록을 통해 OS 불팅 후 로그인 없이 프로그램이 자동으로 실행되도록 설정합니다. Windows 작업 스카에드러, Linux systemd, Docker restart 정책 등을 활용합니다.

## 3. 실행 상태 감심
프로그램을 감심하는 별도 프로세스를 두어 프로세스 상태, CPU/메모리 사용, 로그 오류, 최근 작업 시간, 네트워크/DB 연결 상태를 주기적으로 확인하고, 죽으면 재시작합니다. 복잡 실험 시 안전모드로 진입합니다.

## 4. 오류 발생 시 자동 재시도
오류 발생 시 재시도 구치를 적용합니다: 1차 실험 후 5초, 2차 실험 후 30초, 3차 실험 후 3분. 계속 실험하면 안전모드로 전환합니다.

## 5. 마지막 작업 진행 점 저장
작업 상태를 체크폰트로 저장하여 재불팅 후 마지막 성공 진행 점부터 이어서 진행합니다. 진행 중, 완료, 실험 상태 및 실험 원인, 재시도 횟수 등을 기록합니다.

## 6. 안전모드와 알림
같은 오류가 복중됐다나 세서/장비 상태가 비정상일 때 안전모드로 전환하여 위험 동작을 중진하고 관리자에게 알림합니다.

## 7. 위험 장비 자동 재게 금지
머터, 퍼프, 컨베이어 등 물리 장비는 전원 보관 후 자동으로 재게하지 않고 안전 확인 후에만 재게합니다.
